HCSS

Hyper Cascade Style Sheet

HCSS is not a new CSS language but a “Best-Practices” guide on how to write faster, smarter and more robust CSS. Following this style you will be able to code 70% faster.
  1. 1. It's Hyper
  2. 2. It's ordered
  3. 3. It's reusable

It's Hyper

We all code html like in the following example:

HTML CLassic structure

<a href='www.hcss.com' title='HCSS' rel='no_follow' class='css-class'></a>

If someone were to code HTML like the following example, I’m sure we would say “you crazy man!”

HTML with CSS LIKE Structure

<a
  href=”www.hcss.com”
  title=”HCSS”
  rel=”no_follow”
  class=”css-class”
>

Nonetheless it’s normal for people to code CSS like this

Classical CSS

main-content{
  width: 100px;
  height: 100px;
  margin: 0px 20px;
  color: black;
}

This approach give us abnormally long files were it’s very hard to find the class we were looking for without using the searcher or a ton of comments.

HCSS approach thinks about how HTML is writen and applys it to CSS.

HCSS

main-content{ width:100px; height:100px; margin:0px 20px; color:white; }

Somethin very interesting happens when writting this way. Suddenly, we can use the same indentation style as in HTML, providing hierarchycal identation inside CSS.

It's ordered

HCSS is thought as a coordiante system. First you look on the Y axis. By being indented as the HTMl is much easier to find anything. Once you are on the class your were looking for you need to look on the X axis. In order to make this step faster and easier, a strict order must be used. HCSS is basically ordere in three categories of properties:
dimensions/structural/positioning - background and font style - the rest

Widht - Height / Display / Position / left, right, top, bottom / float / margins / paddings / background / color / text types / CSS3 features (text-shadow/ border radius / box-shadow) / opacity / overflow / z-index
